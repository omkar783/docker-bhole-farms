# Security Policy

## Supported Versions

| Version | Supported          |
|---------|-------------------|
| Latest  | ✅ Supported       |
| Older   | ❌ Not supported   |

Only the latest deployed version of Bhole Farms receives security updates.

## Reporting a Vulnerability

We take security vulnerabilities seriously. Please report them privately.

**Do not** open public GitHub issues for security vulnerabilities.

### How to Report

1. **Email:** `security@bholefarms.com`
2. **PGP Key:** Available on request

### What to Include

- Description of the vulnerability
- Steps to reproduce
- Affected version / commit hash
- Potential impact
- Any suggested fix (optional)

### Response Timeline

| Timeframe | Action |
|-----------|--------|
| 48 hours  | Initial acknowledgment |
| 5 days    | Assessment and severity rating |
| 14 days   | Fix deployed (critical issues) |
| 30 days   | Fix deployed (non-critical issues) |

### After Fix

- We will notify you when the fix is deployed
- We will credit you in the release notes (if desired)
- A CVE may be requested for significant issues

## Bug Bounty

We do not currently operate a bug bounty program.
Vulnerabilities are reported and fixed on a good-faith basis.

## Disclosure Policy

- We will fix the issue before public disclosure
- Coordinated disclosure is preferred (90-day window)
- Public disclosure after fix is deployed and users have updated
